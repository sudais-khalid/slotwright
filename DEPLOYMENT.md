# Deployment

Generated by vibe-ship. Three services, wired together with Docker Compose:

| Service | Image | Port (host) | Notes |
|---|---|---|---|
| `mongo` | `mongo:7` | not exposed | internal network only, named volume, root-auth enabled |
| `backend` | built from `backend/Dockerfile` (Python 3.12 / FastAPI) | `8000` | Swagger UI at `/docs`; also reachable via the frontend's proxy |
| `frontend` | built from `frontend/Dockerfile` (Vite build → nginx) | `8080` | nginx proxies `/api/*` and `/ws/*` to `backend:8000` |

## Run it

```
cp .env.example .env      # fill in MONGO_ROOT_USER / MONGO_ROOT_PASSWORD
docker compose up --build
```

Open http://localhost:8080 for the dashboard, http://localhost:8000/docs for the API.

`real_data/` on the host is bind-mounted straight into the backend container, so
dropping a new timetable/venue/event-plan PDF there (or uploading one from the
dashboard's Data tab) is picked up without a rebuild — only a re-import
(`POST /api/data/use-real`, or the Data tab's button).

## CI/CD

`.github/workflows/ci-cd.yml`: on every push/PR, runs the backend pytest suite
and the frontend production build, plus a non-blocking Trivy dependency scan.
On push to `main`, builds and pushes both images to GHCR
(`ghcr.io/<owner>/<repo>-backend` and `-frontend`) — no extra secrets needed,
it authenticates with the built-in `GITHUB_TOKEN`.

## Manual follow-ups

- **CORS_ORIGINS** in `.env` currently only allows `localhost:8080`. Add your
  real domain before deploying anywhere else.
- **MongoDB credentials**: `.env.example` ships a placeholder password —
  `.env` itself is gitignored, but double check it before any deploy.
- **Resource limits** in `docker-compose.yml` (backend 1.5 CPU/1GB, mongo 1
  CPU/512MB, frontend 0.5 CPU/128MB) are conservative starting points sized
  for tabular Q-Learning training runs — raise the backend's memory limit if
  you train with a much larger Q-table (many more students/venues/semesters).
- **No rate limiting** on the API — fine for a single-user academic demo;
  add one (e.g. `slowapi`) before exposing this publicly.
- **HTTPS/TLS** isn't configured — the compose file is for local/LAN use.
  Put a reverse proxy (Caddy, Traefik, or a cloud load balancer) with a real
  certificate in front of it for anything internet-facing.
- **Training is synchronous per request** — one training run at a time
  (enforced by the backend already); horizontal scaling of the backend
  wouldn't help training throughput without moving the Q-table to shared
  storage, which is out of scope for this project.
